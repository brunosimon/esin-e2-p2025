JavaScript Classes

Maintenant que nous savons comment importer et exporter du code, que devons nous exporter exactement ?

Nous allons exporter des classes qui vont nous permettre de faire du code orienté objet

On parle de OOP

Object Oriented Programmation

Créer une classe

Pour créer une classe, nous pouvons utiliser le code suivant


class HelloClass {}

Rien ne va se produire car nous n'utilisons pas la classe

☝️ Même si nous utilisons des accolades, il ne s'agit pas d'un objet

Par convention, nous nommons les classes en PascalCase

Instancier une classe

Une classe est comme un schéma (blueprint)

Nous pouvons l'utiliser pour créer plusieurs objets

Imaginons que notre classe est un blueprint qui permet de créer des robots


class Robot {}

Nous pouvons ensuite créer un robot à partir de cette classe


const robot = new Robot()
console.log(robot)

La variable robot est ce qu'on appelle une instance de la classe Robot

Et nous pouvons donc créer plusieurs robots à partir de cette classe


class Robot {}

const wallE = new Robot()
const ultron = new Robot()
const astroBoy = new Robot()

Méthodes

C'est un bon début, mais nos robots ne peuvent rien faire

Il est possible d'ajouter des fonctions à la classe

Et nous pouvons donc créer plusieurs robots à partir de cette classe


class Robot
{
    sayHi()
    {
        console.log('Hello!')
    }
}

Et tous les robots auront accès à cette fonction


ultron.sayHi()
astroBoy.sayHi()

☝️

Les fonctions dans les classes sont appelées méthodes

Constructeur

Et si nous forcions nos robots à remercier leur créateur lorsqu'ils sont construits

Pour cela, nous pouvons ajouter une méthodes nommée constructor


class Robot
{
    constructor()
    {
        console.log('Thank you creator')
    }

    sayHi()
    {
        console.log('Hello!')
    }
}

Il est possible d'ajouter des paramètres à cette classes


class Robot
{
    constructor(name)
    {
        console.log(`I am ${name}. Thank you creator`)
    }

    sayHi()
    {
        console.log('Hello!')
    }
}

Et d'envoyer les données lors de l'instanciation


const wallE = new Robot('Wall-E')
const ultron = new Robot('Ultron')
const astroBoy = new Robot('Astro Boy')

Context

Et si nous aimerions que le robot fournisse son nom lorsqu'il dit "bonjour"

Nous ne voulons pas avoir à fournir le nom à chaque fois que l'on appelle la fonction sayHi

Comme nous fournissons le nom dans le constructor, nous allons faire en sorte de le sauvegarder au sein de l'instance

Pour cela, nous allons utiliser le mot clé this


class Robot
{
    constructor(name)
    {
        this.name = name

        console.log(`I am ${this.name}. Thank you creator`)
    }

    sayHi()
    {
        console.log('Hello!')
    }
}

this est ce qu'on appelle le context

Le context correspond à l'instance et est particulier à chaque instance même si le code est le même dans la classe

☝️

name est ce qu'on appelle une propriété de la classe

Nous pouvons ajouter d'autres propriétés


class Robot
{
    constructor(name, legs)
    {
        this.name = name
        this.legs = legs

        console.log(`I am ${this.name}. Thank you creator`)
    }

    // ...
}

const wallE = new Robot('Wall-E', 0)
const ultron = new Robot('Ultron', 2)
const astroBoy = new Robot('Astro Boy', 2)

Et nous pouvons y accéder depuis l'extérieur de la classe


console.log(wallE.legs)

Les méthodes sont aussi accessibles depuis le context


class Robot
{
    constructor(name, legs)
    {
        this.name = name
        this.legs = legs

        this.sayHi()

        console.log(`I am ${this.name}. Thank you creator`)
    }

    // ...
}

Héritage

L'héritage revient à créer une classe basée sur une autre classe

Nous créons un blueprint à partir d'un autre blueprint

Toutes les méthodes de la classe de base seront disponibles dans la nouvelle classe

Pour illustrer cela, nous allons créer une classe de robots volants basée sur la classe de robots

Créez une classe FlyingRobot basée sur la classe Robot avec le mot clé extends


class FlyingRobot extends Robot
{
    
}

Nous pouvons maintenant l'utiliser sur les robots qui volent


const wallE = new Robot('Wall-E', 0)
const ultron = new FlyingRobot('Ultron', 2)
const astroBoy = new FlyingRobot('Astro Boy', 2)

Actuellement, la classe FlyingRobot n'apporte rien de plus que Robot, mais nous pouvons ajouter de nouvelles méthodes


class FlyingRobot extends Robot
{
    takeOff()
    {
        console.log(`Have a good flight ${this.name}`)
    }

    land()
    {
        console.log(`Welcome back ${this.name}`)
    }
}

Les robots instanciés depuis FlyingRobot peuvent maintenant décoler et atterir


astroBoy.sayHi()
astroBoy.takeOff()
astroBoy.land()

Et ils peuvent toujours dire bonjour

Si nous utilisons le même nom de méthode, cela aura pour effet de l'écraser


class FlyingRobot extends Robot
{
    sayHi()
    {
        console.log(`Hello! My name is ${this.name} and I am a flying robot`)
    }

    // ...
}

Mais il est aussi possible d'appeler la méthode de la classe de base avec le mot clé super


class FlyingRobot extends Robot
{
    sayHi()
    {
        super.sayHi()
        console.log('and I am a flying robot')
    }

    // ...
}

Cela fonctionne aussi avec la méthode constructor sauf qu'il faut appeler super() sans préciser de méthode


class FlyingRobot extends Robot
{
    constructor(name, legs)
    {
        super(name, legs)

        this.canFly = true
    }

    // ...
}

☝️

En réalité, nous sommes obligés d'appeler super() dans le constructor d'une classe héritée

Modules

Nous pouvons utiliser les modules pour exporter nos classes

Créez un fichier src/Robot.js qui export la classe Robot


export default class Robot
{
    constructor(name, legs)
    {
        this.name = name
        this.legs = legs

        console.log(`I am ${this.name}. Thank you creator`)

        this.sayHi()
    }

    sayHi()
    {
        console.log(`Hello! My name is ${this.name}`)
    }
}

Créez un fichier src/FlyingRobot.js qui export la classe FlyingRobot


export default class FlyingRobot extends Robot
{
    constructor(name, legs)
    {
        super(name, legs)

        this.canFly = true
    }

    sayHi()
    {
        console.log(`Hello! My name is ${this.name} and I'm a flying robot`)
    }

    takeOff()
    {
        console.log(`Have a good flight ${this.name}`)
    }

    land()
    {
        console.log(`Welcome back ${this.name}`)
    }
}

⚠️

Il ne faut pas oublier d'importer la classe Robot dans le fichier src/FlyingRobot.js


import Robot from './Robot.js'

export default class FlyingRobot extends Robot
{
    // ...

Dans src/index.js, nous pouvons maintenant importer et utiliser nos classes


import Robot from './Robot.js'
import FlyingRobot from './FlyingRobot.js'

const wallE = new Robot('Wall-E', 0)
const ultron = new FlyingRobot('Ultron', 2)
const astroBoy = new FlyingRobot('Astro Boy', 2)

Conclusion

Les classes peuvent sembler difficiles, mais elles rendent le code beaucoup plus maintenable et réutilisable

Il n'est pas rare de voir des applications 100% orientées objet
